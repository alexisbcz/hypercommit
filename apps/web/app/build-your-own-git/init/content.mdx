A Git repository is, at its core, a directory called `.git` next to your working tree. `git init` creates that directory and seeds it with the minimum structure Git needs to start tracking changes.

In this lesson you'll write `git init` in Go, but we'll do it test-first. Each test is a tiny, observable promise about what `mygit init` does. We add the test, watch it fail, then make it pass with the smallest change we can. Twelve rounds in, you'll have a working `init` that real Git accepts as one of its own.

## Why test-first?

`init` is a *boundary* command. It doesn't compute anything interesting; it talks to the filesystem. The only honest way to verify it is to run it and check what landed on disk. That's exactly what tests do. Writing the test first frames the work, every line of code we add exists to flip a red bar to green.

The spec is also decentralized. It lives partly in the Git docs, partly in `setup.c`, partly in habits real users have built around `.git/HEAD`. Tests pin down which slice of that spec we're committing to. When we ship, anyone reading the test file knows what "done" means.

## What `git init` actually does

Run real Git on an empty folder and look:

```bash
mkdir demo && cd demo
git init
ls .git
```

<DirectoryTree
  paths={[
    ".git/HEAD",
    ".git/config",
    ".git/description",
    ".git/hooks/",
    ".git/info/",
    ".git/objects/",
    ".git/refs/",
  ]}
/>

Five entries are mandatory; the rest are convenience.

| Entry         | Purpose                                                     |
| ------------- | ----------------------------------------------------------- |
| `HEAD`        | Pointer to the current branch                               |
| `objects/`    | Content-addressable storage for blobs, trees, commits, tags |
| `refs/heads/` | Branch tips                                                 |
| `refs/tags/`  | Tag refs                                                    |
| `config`      | Repository-local configuration                              |

`description`, `hooks/`, `info/exclude`, Git works without them. We'll skip them.

## The test harness

Tests will exec the compiled binary and inspect what it wrote. That's the most honest check we can run, and it doesn't require any internal hooks into our code.

```go title="init_test.go"
package main_test

import (
	"bytes"
	"os"
	"os/exec"
	"path/filepath"
	"strings"
	"testing"
)

var binPath string

func TestMain(m *testing.M) {
	dir, err := os.MkdirTemp("", "mygit-bin-")
	if err != nil {
		panic(err)
	}
	binPath = filepath.Join(dir, "mygit")
	build := exec.Command("go", "build", "-o", binPath, ".")
	build.Stderr = os.Stderr
	if err := build.Run(); err != nil {
		panic(err)
	}
	code := m.Run()
	_ = os.RemoveAll(dir)
	os.Exit(code)
}

type result struct {
	stdout, stderr string
	err            error
}

func run(t *testing.T, workdir string, args ...string) result {
	t.Helper()
	var stdout, stderr bytes.Buffer
	cmd := exec.Command(binPath, args...)
	cmd.Dir = workdir
	cmd.Stdout = &stdout
	cmd.Stderr = &stderr
	err := cmd.Run()
	return result{stdout: stdout.String(), stderr: stderr.String(), err: err}
}

func mustInit(t *testing.T, workdir string, args ...string) result {
	t.Helper()
	res := run(t, workdir, append([]string{"init"}, args...)...)
	if res.err != nil {
		t.Fatalf("mygit init %v failed: %v\nstdout: %s\nstderr: %s",
			args, res.err, res.stdout, res.stderr)
	}
	return res
}
```

Three things to notice:

- **`TestMain` builds the binary once** into a tempdir and reuses it for every test. Building per-test would dominate runtime.
- **`t.TempDir()`** gives each test a clean working directory that's automatically deleted afterward. We never touch the user's filesystem.
- **`mustInit`** lifts the "command must succeed" assertion out of every test body. When something goes wrong, it dumps stdout *and* stderr, exactly what you want for a CLI test that just failed in CI.

`go test ./...` should pass with no tests yet. Now we play.

## Round 1: `.git` exists

The smallest possible promise:

```go title="init_test.go (add)"
func TestInit_CreatesGitDirectory(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir)
	info, err := os.Stat(filepath.Join(dir, ".git"))
	if err != nil {
		t.Fatalf(".git not created: %v", err)
	}
	if !info.IsDir() {
		t.Fatal(".git exists but is not a directory")
	}
}
```

Watch it fail:

```text
--- FAIL: TestInit_CreatesGitDirectory
    .git not created: stat /.../mygit-test/.git: no such file or directory
```

The smallest fix: create the directory.

```go title="init.go (Action body)"
import "os"

Action: func(ctx context.Context, cmd *cli.Command) error {
	return os.MkdirAll(".git", 0o755)
},
```

`Action` runs in whatever directory the user shelled into. `os.MkdirAll(".git", ...)` resolves relative to that working directory, exactly what real `git init` does. Test passes.

## Round 2: objects, refs/heads, refs/tags

Git refuses to call a directory a repository unless it has the canonical layout. Lock it in:

```go title="init_test.go (add)"
func TestInit_CreatesObjectsAndRefs(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir)
	for _, sub := range []string{
		".git/objects",
		".git/refs/heads",
		".git/refs/tags",
	} {
		info, err := os.Stat(filepath.Join(dir, sub))
		if err != nil {
			t.Errorf("%s missing: %v", sub, err)
			continue
		}
		if !info.IsDir() {
			t.Errorf("%s is not a directory", sub)
		}
	}
}
```

The whole `Action` becomes a loop over the layout. Lift it into a function, it's about to grow.

```go title="init.go"
import (
	"fmt"
	"path/filepath"
)

Action: func(ctx context.Context, cmd *cli.Command) error {
	return runInit()
},

func runInit() error {
	gitDir := ".git"
	for _, d := range []string{
		gitDir,
		filepath.Join(gitDir, "objects"),
		filepath.Join(gitDir, "refs", "heads"),
		filepath.Join(gitDir, "refs", "tags"),
	} {
		if err := os.MkdirAll(d, 0o755); err != nil {
			return fmt.Errorf("create %s: %w", d, err)
		}
	}
	return nil
}
```

Two tests green. The layout lives in one slice, single source of truth, easy to extend.

## Round 3: HEAD points at `main`

Without `HEAD`, no Git client (real or otherwise) considers the directory a repo. Pin its content exactly:

```go title="init_test.go (add)"
func TestInit_HEADPointsAtMain(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir)
	got, err := os.ReadFile(filepath.Join(dir, ".git", "HEAD"))
	if err != nil {
		t.Fatal(err)
	}
	want := "ref: refs/heads/main\n"
	if string(got) != want {
		t.Errorf("HEAD = %q, want %q", string(got), want)
	}
}
```

Three details that bite if you skip them:

- **Trailing newline.** Real Git writes one. Some readers tolerate its absence, some don't. Match Git.
- **The `ref:` prefix.** HEAD is a *symbolic ref*, it names another ref instead of holding a SHA directly.
- **`main`, not `master`.** As of Git 2.30 the default branch name is configurable; `main` has been the community default since 2020.

Add the file write to `runInit`:

```go title="init.go (additions)"
headPath := filepath.Join(gitDir, "HEAD")
if err := os.WriteFile(headPath, []byte("ref: refs/heads/main\n"), 0o644); err != nil {
	return fmt.Errorf("write HEAD: %w", err)
}
```

Three green.

## Round 4: `config` with a `[core]` section

Git reads config through `git_config_get_int("core.repositoryformatversion", ...)`. If the file is missing or that key isn't `0`, it bails. Encode the contract:

```go title="init_test.go (add)"
func TestInit_ConfigHasCoreSection(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir)
	cfg, err := os.ReadFile(filepath.Join(dir, ".git", "config"))
	if err != nil {
		t.Fatal(err)
	}
	s := string(cfg)
	for _, want := range []string{
		"[core]",
		"repositoryformatversion = 0",
		"filemode = true",
		"bare = false",
		"logallrefupdates = true",
	} {
		if !strings.Contains(s, want) {
			t.Errorf("config missing %q\nfull config:\n%s", want, s)
		}
	}
}
```

We test *substrings*, not exact bytes. Whitespace, key order, and trailing newlines vary across real Git versions; the meaning doesn't.

Add the constant and write:

```go title="init.go (additions)"
const defaultConfigContent = `[core]
	repositoryformatversion = 0
	filemode = true
	bare = false
	logallrefupdates = true
`

// inside runInit, after writing HEAD:
configPath := filepath.Join(gitDir, "config")
if err := os.WriteFile(configPath, []byte(defaultConfigContent), 0o644); err != nil {
	return fmt.Errorf("write config: %w", err)
}
```

The indentation inside `defaultConfigContent` should be tab characters; Git accepts spaces too, but tabs match what real Git produces. Your editor's "indent with spaces" setting will silently bite you here, verify with `cat -A`.

Four green.

## Round 5: the success message

`git init`'s output is parseable. CI scripts grep for the exact prefix; so will our test:

```go title="init_test.go (add)"
func TestInit_PrintsInitializedMessage(t *testing.T) {
	dir := t.TempDir()
	res := mustInit(t, dir)
	abs, err := filepath.EvalSymlinks(filepath.Join(dir, ".git"))
	if err != nil {
		t.Fatal(err)
	}
	want := "Initialized empty Git repository in " + abs + "/\n"
	if res.stdout != want {
		t.Errorf("stdout = %q, want %q", res.stdout, want)
	}
}
```

The trailing slash matters. Real Git appends one if the path doesn't already end in `/`, so the resolved path always does. The `EvalSymlinks` call is a macOS quirk, `/var/folders/...` is a symlink to `/private/var/folders/...`, and we want both sides of the comparison to live in the same world.

```go title="init.go (additions)"
abs, err := filepath.Abs(gitDir)
if err != nil {
	abs = gitDir
}
if resolved, err := filepath.EvalSymlinks(abs); err == nil {
	abs = resolved
}
fmt.Fprintf(cmd.Writer, "Initialized empty Git repository in %s/\n", abs)
```

Use `cmd.Writer` (urfave/cli's stdout abstraction) instead of `fmt.Println`, it's the seam we'll need when we write `--quiet` later.

Five green.

## Round 6: reinit detection

Run `git init` twice in the same place and the message changes:

> Reinitialized existing Git repository in /path/to/.git/

```go title="init_test.go (add)"
func TestInit_ReinitMessage(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir)
	res := mustInit(t, dir)
	if !strings.HasPrefix(res.stdout, "Reinitialized existing Git repository in ") {
		t.Errorf("expected reinit message, got: %s", res.stdout)
	}
}
```

The honest signal that "this is already a repository" is an existing HEAD file. Detect *before* we (re)write anything, otherwise we'd see the HEAD we just wrote:

```go title="init.go (additions)"
reinit := false
if _, err := os.Stat(filepath.Join(gitDir, "HEAD")); err == nil {
	reinit = true
}

// ...mkdirs and writes...

msg := "Initialized empty Git repository"
if reinit {
	msg = "Reinitialized existing Git repository"
}
fmt.Fprintf(cmd.Writer, "%s in %s/\n", msg, abs)
```

Six green.

## Round 7: don't clobber a user-modified HEAD

If someone has switched branches in this repo by editing `HEAD`, a re-`init` shouldn't reset them. Encode it:

```go title="init_test.go (add)"
func TestInit_PreservesUserModifiedHEAD(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir)
	custom := "ref: refs/heads/dev\n"
	headPath := filepath.Join(dir, ".git", "HEAD")
	if err := os.WriteFile(headPath, []byte(custom), 0o644); err != nil {
		t.Fatal(err)
	}
	mustInit(t, dir)
	got, err := os.ReadFile(headPath)
	if err != nil {
		t.Fatal(err)
	}
	if string(got) != custom {
		t.Errorf("reinit overwrote HEAD: got %q, want %q", string(got), custom)
	}
}
```

Two-line fix: only write if absent.

```go title="init.go (replace HEAD write)"
import "errors"

if _, err := os.Stat(headPath); errors.Is(err, os.ErrNotExist) {
	if err := os.WriteFile(headPath, []byte("ref: refs/heads/main\n"), 0o644); err != nil {
		return fmt.Errorf("write HEAD: %w", err)
	}
}
```

Apply the same write-if-absent pattern to `config`. Same reason: a user may have set `user.email` in repo config; we shouldn't blow it away.

Seven green.

## Round 8: optional `<directory>` argument

Real Git: `git init my-repo` creates `my-repo/.git`. Ours should too.

```go title="init_test.go (add)"
func TestInit_DirectoryArgument(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir, "myrepo")
	if _, err := os.Stat(filepath.Join(dir, "myrepo", ".git", "HEAD")); err != nil {
		t.Fatal(err)
	}
}
```

`runInit` needs to know about a target directory. Pass options in instead of using globals:

```go title="init.go"
type initOptions struct {
	target string
}

Action: func(ctx context.Context, cmd *cli.Command) error {
	target := "."
	if cmd.NArg() > 0 {
		target = cmd.Args().Get(0)
	}
	return runInit(initOptions{target: target}, cmd.Writer)
},

func runInit(opts initOptions, stdout io.Writer) error {
	if err := os.MkdirAll(opts.target, 0o755); err != nil {
		return fmt.Errorf("create %s: %w", opts.target, err)
	}
	gitDir := filepath.Join(opts.target, ".git")
	// ...rest unchanged, using gitDir and stdout
}
```

We `MkdirAll(opts.target, ...)` first, `git init nested/path/to/repo` should create the intermediate directories.

Eight green.

## Round 10: `--bare`

A bare repository has no working tree: the contents of `.git` live at the top level, and `core.bare = true`.

```go title="init_test.go (add)"
func TestInit_BareFlag(t *testing.T) {
	dir := t.TempDir()
	mustInit(t, dir, "--bare")
	if _, err := os.Stat(filepath.Join(dir, ".git")); !os.IsNotExist(err) {
		t.Errorf("bare repo should not have .git/ subdirectory")
	}
	if _, err := os.Stat(filepath.Join(dir, "HEAD")); err != nil {
		t.Fatal("bare repo missing HEAD at root:", err)
	}
	if _, err := os.Stat(filepath.Join(dir, "objects")); err != nil {
		t.Fatal("bare repo missing objects/ at root:", err)
	}
	cfg, err := os.ReadFile(filepath.Join(dir, "config"))
	if err != nil {
		t.Fatal(err)
	}
	if !strings.Contains(string(cfg), "bare = true") {
		t.Errorf("bare config missing bare = true:\n%s", cfg)
	}
}
```

If our code is shaped right, this is two-line surgery, `gitDir` becomes the target directly, and we pick a different `config` template:

```go title="init.go"
const bareConfigContent = `[core]
	repositoryformatversion = 0
	filemode = true
	bare = true
`

// flags:
&cli.BoolFlag{Name: "bare", Usage: "create a bare repository"},

// in Action, into initOptions:
bare: cmd.Bool("bare"),

// in runInit:
gitDir := filepath.Join(opts.target, ".git")
if opts.bare {
	gitDir = opts.target
}

// when writing config:
content := defaultConfigContent
if opts.bare {
	content = bareConfigContent
}
```

Ten green.

## Round 12: the acid test

A repo isn't a repo until real Git agrees:

```go title="init_test.go (add)"
func TestInit_AcceptedByRealGit(t *testing.T) {
	if _, err := exec.LookPath("git"); err != nil {
		t.Skip("real git not available")
	}
	dir := t.TempDir()
	mustInit(t, dir)
	cmd := exec.Command("git", "-C", dir, "status")
	cmd.Env = append(os.Environ(),
		"GIT_CONFIG_GLOBAL=/dev/null",
		"GIT_CONFIG_SYSTEM=/dev/null",
	)
	out, err := cmd.CombinedOutput()
	if err != nil {
		t.Fatalf("real git rejected our repo: %v\n%s", err, out)
	}
	if !strings.Contains(string(out), "On branch main") {
		t.Errorf("unexpected git status output:\n%s", out)
	}
}
```

We override `GIT_CONFIG_GLOBAL` and `GIT_CONFIG_SYSTEM` so the test isn't sensitive to your machine's config, your global `init.defaultBranch = trunk` would otherwise sneak in.

If everything earlier was right, this passes immediately. No code change needed.

Twelve green. We've shipped a Git-compatible `init`.

## The final source

For reference, here's the full `init.go` after twelve rounds, every line earned a green test.

<CollapsibleCode filename="init.go" caption="full source after Round 12">

```go
package main

import (
	"context"
	"errors"
	"fmt"
	"io"
	"os"
	"path/filepath"

	"github.com/urfave/cli/v3"
)

const defaultConfigContent = `[core]
	repositoryformatversion = 0
	filemode = true
	bare = false
	logallrefupdates = true
`

const bareConfigContent = `[core]
	repositoryformatversion = 0
	filemode = true
	bare = true
`

func initCommand() *cli.Command {
	return &cli.Command{
		Name:      "init",
		Usage:     "create an empty Git repository",
		ArgsUsage: "[directory]",
		Flags: []cli.Flag{
			&cli.StringFlag{
				Name:    "initial-branch",
				Aliases: []string{"b"},
				Usage:   "override the name of the initial branch",
				Value:   "main",
			},
			&cli.BoolFlag{Name: "bare", Usage: "create a bare repository"},
			&cli.BoolFlag{Name: "quiet", Aliases: []string{"q"}, Usage: "be quiet"},
		},
		Action: func(ctx context.Context, cmd *cli.Command) error {
			target := "."
			if cmd.NArg() > 0 {
				target = cmd.Args().Get(0)
			}
			return runInit(initOptions{
				target:        target,
				bare:          cmd.Bool("bare"),
				quiet:         cmd.Bool("quiet"),
				initialBranch: cmd.String("initial-branch"),
			}, cmd.Writer)
		},
	}
}

type initOptions struct {
	target        string
	bare          bool
	quiet         bool
	initialBranch string
}

func runInit(opts initOptions, stdout io.Writer) error {
	if stdout == nil {
		stdout = os.Stdout
	}
	if err := os.MkdirAll(opts.target, 0o755); err != nil {
		return fmt.Errorf("create %s: %w", opts.target, err)
	}

	gitDir := filepath.Join(opts.target, ".git")
	if opts.bare {
		gitDir = opts.target
	}

	reinit := false
	if _, err := os.Stat(filepath.Join(gitDir, "HEAD")); err == nil {
		reinit = true
	}

	for _, d := range []string{
		gitDir,
		filepath.Join(gitDir, "objects"),
		filepath.Join(gitDir, "refs", "heads"),
		filepath.Join(gitDir, "refs", "tags"),
	} {
		if err := os.MkdirAll(d, 0o755); err != nil {
			return fmt.Errorf("create %s: %w", d, err)
		}
	}

	headPath := filepath.Join(gitDir, "HEAD")
	if _, err := os.Stat(headPath); errors.Is(err, os.ErrNotExist) {
		head := fmt.Sprintf("ref: refs/heads/%s\n", opts.initialBranch)
		if err := os.WriteFile(headPath, []byte(head), 0o644); err != nil {
			return fmt.Errorf("write HEAD: %w", err)
		}
	}

	configPath := filepath.Join(gitDir, "config")
	if _, err := os.Stat(configPath); errors.Is(err, os.ErrNotExist) {
		content := defaultConfigContent
		if opts.bare {
			content = bareConfigContent
		}
		if err := os.WriteFile(configPath, []byte(content), 0o644); err != nil {
			return fmt.Errorf("write config: %w", err)
		}
	}

	if !opts.quiet {
		abs, err := filepath.Abs(gitDir)
		if err != nil {
			abs = gitDir
		}
		if resolved, err := filepath.EvalSymlinks(abs); err == nil {
			abs = resolved
		}
		msg := "Initialized empty Git repository"
		if reinit {
			msg = "Reinitialized existing Git repository"
		}
		fmt.Fprintf(stdout, "%s in %s/\n", msg, abs)
	}
	return nil
}
```

            int init_shared_repository, unsigned int flags)

    safe_create_dir(the_repository, git_dir, 0);

    reinit = create_default_files(template_dir, original_git_dir,

## What we skipped (on purpose)

1. **Template directory.** Git copies hook samples and `info/exclude` from `/usr/share/git-core/templates`. Optional polish.
2. **`--separate-git-dir`.** Lets the `.git` directory live somewhere else and leaves only a `.git` *file* (with `gitdir: <path>`) in the working tree. Two-line follow-up.
3. **`--shared`.** Sets the umask of created files so multiple users on the same box can write into the repo. Important for shared servers, irrelevant for our tutorial.
4. **`--object-format` / `--ref-format`.** Hash-algorithm and ref-storage selection. Both belong to a later lesson.
